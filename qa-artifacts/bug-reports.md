## BUG-01. PPT 슬라이드 수 입력 필드 overflow 시 `Infinity` 출력 및 입력 필드 복구 불가

| 항목 | 내용 |
|---|---|
| 기능 | PPT 생성 |
| 관련 TC | `TC_066`, `TC_067`, `TC_073` |
| 우선순위 | P2 |
| 상태 | TODO |
| 발생 조건 | PPT 생성 화면의 슬라이드 수 입력 필드에 매우 긴 숫자 입력 |
| 기대 결과 | 입력값이 `Infinity`로 변환되지 않고, 유효성 에러 또는 입력 제한으로 제어되어야 함 |
| 실제 결과 | 입력값이 `Infinity`로 출력되며 입력 필드 복구가 어려움 |
| 영향 | 사용자가 의도하지 않은 숫자 값으로 PPT 생성 요청을 보낼 수 있음 |
| 재현 절차 | PPT 생성 화면 진입 -> 슬라이드 수 입력 필드에 긴 숫자 입력 -> 입력 필드 표시값 확인 |
| 회귀 관리 | PPT 숫자 필드 긴 숫자 입력 검증 테스트를 `pytest.mark.xfail`로 유지 |

| 발생 조건 | PPT 생성 화면의 주제 입력 필드에 공백만 입력 |
